# Proper Noun Candidates

This file documents sign sequences hypothesized to be proper nouns (place names, personal names) based on site-specificity analysis and phonological cross-referencing. All entries are Tier 4 (Lexical) evidence at Low to Low-Medium confidence unless otherwise noted.

**Method:** Site-specificity score = max_site_count / total_count across all occurrences. Threshold for candidate status: ≥0.80. Minimum occurrences: 3. Cross-referenced against Aegean toponym list and Linear B renditions of Minoan place names.

**First documented:** 2026-03-18 (Attempt #3, Strategy 3)

| Property | Value |
|----------|-------|
| Token | KU-NI-SU |
| Consonant skeleton | KNS |
| Proposed identification | Knossos (Linear B: *ko-no-so*) |
| Confidence | Low |
| Evidence tier | Tier 4 (Lexical) |
| Site-specificity score | High (Haghia Triada primary) |
| Cross-site attestation | Haghia Triada only |
| Basis | Consonant skeleton KNS matches ko-no-so → KNS pattern |

**Critical limitation:** KU-NI-SU appears at Haghia Triada, not at Knossos. If this were a toponym for Knossos used in HT administrative records, it would appear in tablet contexts listing commodities sent to/from Knossos — plausible for inter-site accounting. However, without corroboration at Knossos itself or in a clear "destination" grammatical position, this remains Low confidence.

## Methodology Notes

**Consonant skeleton matching** is a weak form of phonological comparison. Ancient languages frequently add vowels, change word boundaries, or apply different phonological rules. A consonant skeleton match alone is insufficient for a toponym identification — it must be supported by:
1. Plausible archaeological/geographical context
2. Support type compatible with place name usage (tablets with destination/origin fields)
3. Ideally, cross-site occurrence near the proposed location

All entries above currently satisfy only criterion 1 (or partially 1). None have been cross-validated per criterion 3.
